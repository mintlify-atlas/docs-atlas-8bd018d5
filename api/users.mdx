## Overview

The Users API allows administrators to manage operator and admin accounts in the system. All endpoints require admin role authentication.

<Warning>
All endpoints in this group require **admin role**. Operators cannot access these endpoints.
</Warning>

## Authentication

All requests must include a valid JWT token with `admin` role:

```bash
Authorization: Bearer <admin_jwt_token>
```

## User Roles

- **operator** - Can view and manage divorce cases, but cannot manage users
- **admin** - Full system access including user management

## List Users

<CodeGroup>
```bash curl
curl -X GET "http://localhost:8000/api/users/?include_inactive=false" \
```

**GET** `/api/users/`

Lists all users in the system.

### Query Parameters

<ParamField query="include_inactive" type="boolean" default="false">
  If true, includes deactivated users in the results
</ParamField>

### Response

<ResponseField name="[]" type="array">
  Array of user objects
  
  <Expandable title="User object properties">
    <ResponseField name="id" type="integer">
      Unique user identifier
    </ResponseField>
    
    <ResponseField name="username" type="string">
      Username for login
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User email address
    </ResponseField>
    
    <ResponseField name="full_name" type="string">
      Full name (optional)
    </ResponseField>
    
    <ResponseField name="role" type="string">
      User role: "operator" or "admin"
    </ResponseField>
    
    <ResponseField name="is_active" type="boolean">
      Whether the user account is active
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of account creation
    </ResponseField>
  </Expandable>
</ResponseField>

## Get User Details

**GET** `/api/users/{user_id}`

Retrieves detailed information about a specific user.

### Path Parameters

<ParamField path="user_id" type="integer" required>
  The ID of the user to retrieve
</ParamField>

### Response

Returns a single user object with the same structure as the list endpoint.

### Error Responses

- **404 Not Found** - User does not exist
- **403 Forbidden** - Requires admin role

## Create User

**POST** `/api/users/`

Creates a new operator or admin account.

### Request Body

<ParamField body="username" type="string" required>
  Unique username for login (must be unique)
</ParamField>

<ParamField body="email" type="string" required>
  User email address (must be unique and valid)
</ParamField>

<ParamField body="password" type="string" required>
  Password (minimum 6 characters)
</ParamField>

<ParamField body="full_name" type="string">
  User's full name (optional)
</ParamField>

<ParamField body="role" type="string" default="operator">
  User role: "operator" or "admin"
</ParamField>

### Example Request

```json
{
  "username": "carlos.ruiz",
  "email": "cruiz@defensoria.gob.ar",
  "password": "SecurePass123!",
  "full_name": "Carlos Ruiz",
  "role": "operator"
}
```

### Response

Returns the created user object (password excluded).

### Validation Rules

- Username must be unique
- Email must be unique and valid format
- Password must be at least 6 characters
- Role must be either "operator" or "admin"

### Error Responses

- **400 Bad Request** - Validation error (duplicate username/email, invalid role, short password)
- **403 Forbidden** - Requires admin role

## Update User

**PUT** `/api/users/{user_id}`

Updates an existing user's information.

### Path Parameters

<ParamField path="user_id" type="integer" required>
  The ID of the user to update
</ParamField>

### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="email" type="string">
  New email address (must be unique)
</ParamField>

<ParamField body="full_name" type="string">
  Updated full name
</ParamField>

<ParamField body="role" type="string">
  Updated role: "operator" or "admin"
</ParamField>

<ParamField body="is_active" type="boolean">
  Account status (true = active, false = deactivated)
</ParamField>

<Note>
Username cannot be changed after account creation.
</Note>

### Example Request

```json
{
  "email": "carlos.ruiz.new@defensoria.gob.ar",
  "full_name": "Carlos Alberto Ruiz",
  "is_active": true
}
```

### Safety Features

- Admins cannot change their own role to operator
- Admins cannot deactivate their own account
- Email uniqueness is enforced

### Error Responses

- **400 Bad Request** - Self-demotion attempt, email already in use, invalid role
- **403 Forbidden** - Requires admin role
- **404 Not Found** - User does not exist

## Delete User

**DELETE** `/api/users/{user_id}`

Permanently deletes a user account.

### Path Parameters

<ParamField path="user_id" type="integer" required>
  The ID of the user to delete
</ParamField>

### Response

Returns `204 No Content` on success.

<Warning>
This action is **permanent** and cannot be undone. All user data will be lost.
</Warning>

### Safety Features

- Admins cannot delete their own account

### Error Responses

- **400 Bad Request** - Attempting to delete own account
- **403 Forbidden** - Requires admin role
- **404 Not Found** - User does not exist

## Change User Password

**POST** `/api/users/{user_id}/change-password`

Changes a user's password (admin can reset any user's password).

### Path Parameters

<ParamField path="user_id" type="integer" required>
  The ID of the user whose password will be changed
</ParamField>

### Request Body

<ParamField body="new_password" type="string" required>
  New password (minimum 6 characters)
</ParamField>

### Example Request

```json
{
  "new_password": "NewSecurePass456!"
}
```

### Response

```json
{
  "message": "Contraseña actualizada correctamente"
}
```

### Error Responses

- **400 Bad Request** - Password too short (< 6 characters)
- **403 Forbidden** - Requires admin role
- **404 Not Found** - User does not exist
