## Overview

The validation services provide robust input validation and hallucination detection for the divorce platform. These services ensure data quality, detect malicious input, and verify LLM response accuracy.

## Services Covered

1. **SimpleResponseValidationService** - General user response validation
2. **SimpleAddressValidationService** - Address parsing and jurisdictional validation
3. **SimpleDateValidationService** - Date format and age/timeline validation
4. **HallucinationDetectionService** - LLM response hallucination detection

### Utility Functions

```python
DATE_FMT = "%d/%m/%Y"

def parse_date(date_str: str) -> date | None:
    try:
        return datetime.strptime(date_str.strip(), DATE_FMT).date()
    except Exception:
        return None

def years_between(a: date, b: date) -> int:
    return b.year - a.year - ((b.month, b.day) < (a.month, a.day))
```

### Methods

#### validate_birth_date()

```python
def validate_birth_date(self, date_str: str) -> DateValidationResult
```

**Validation Rules:**
1. Format must be `DD/MM/YYYY`
2. Date cannot be in the future
3. Year must be >= 1900
4. Person must be at least 18 years old

**Implementation:**

```python
# date_validation_service_impl.py:17-31
def validate_birth_date(self, date_str: str) -> DateValidationResult:
    d = parse_date(date_str)
    errors = []
    if not d:
        errors.append("Formato de fecha inválido. Use DD/MM/AAAA.")
        return DateValidationResult(False, errors)
    today = date.today()
    if d > today:
        errors.append("La fecha no puede ser futura.")
    if d.year < 1900:
        errors.append("La fecha es demasiado antigua.")
    age = years_between(d, today)
    if age < 18:
        errors.append("Debe ser mayor de 18 años.")
    return DateValidationResult(
        len(errors) == 0, 
        errors, 
        normalized_date=d.strftime(DATE_FMT), 
        age_years=age
    )
```

```python
# date_validation_service_impl.py:33-51
def validate_marriage_date(self, date_str: str, birth_date_solicitante: str, birth_date_conyuge: str):
    d = parse_date(date_str)
    errors = []
    if not d:
        errors.append("Formato de fecha inválido. Use DD/MM/AAAA.")
        return DateValidationResult(False, errors)
    today = date.today()
    if d > today:
        errors.append("La fecha de matrimonio no puede ser futura.")
    b1 = parse_date(birth_date_solicitante)
    b2 = parse_date(birth_date_conyuge)
    if not b1 or not b2:
        errors.append("Fechas de nacimiento inválidas.")
        return DateValidationResult(False, errors)
    if d <= b1 or d <= b2:
        errors.append("La fecha de matrimonio debe ser posterior a las fechas de nacimiento.")
    if years_between(b1, d) < 18 or years_between(b2, d) < 18:
        errors.append("Ambas partes deben tener al menos 18 años al casarse.")
    return DateValidationResult(len(errors) == 0, errors, normalized_date=d.strftime(DATE_FMT))
```

## HallucinationDetectionService

**Location:** `backend/src/application/services/hallucination_detection_service.py:17-320`

Detects hallucinations in LLM-generated responses using rule-based heuristics and optional LLM-based semantic validation.

### Class Definition

```python
class HallucinationDetectionService:
    """
    Servicio para detectar alucinaciones en respuestas del LLM.
    
    Utiliza dos estrategias:
    1. Validación basada en reglas (patterns, heurísticas)
    2. Validación con LLM especializado (glm-4.6:cloud) para análisis semántico
    """
    
    def __init__(self, llm_router=None):
        self.compiled_patterns = [re.compile(p, re.IGNORECASE) for p in self.HALLUCINATION_PATTERNS]
        self.data_patterns = [re.compile(p, re.IGNORECASE) for p in self.SPECIFIC_DATA_PATTERNS]
        self.llm_router = llm_router
        self.use_llm_validation = llm_router is not None
```

### Detection Patterns

#### Hallucination Indicators

```python
HALLUCINATION_PATTERNS = [
    r"según mi base de datos",
    r"en mi sistema",
    r"tengo acceso a",
    r"puedo ver que",
    r"según mis registros",
    r"he verificado que",
]
```

#### Uncertainty Indicators (Good)

```python
UNCERTAINTY_INDICATORS = [
    "posible", "probable", "podría", "quizás", "tal vez",
    "en general", "usualmente", "depende", "puede variar"
]
```

#### Specific Data Patterns (Should Not Invent)

```python
SPECIFIC_DATA_PATTERNS = [
    r"\d{7,8}",                      # DNI
    r"\d{2}/\d{2}/\d{4}",           # Fechas específicas
    r"expediente\s+n[úu]mero\s+\d+", # Números de expediente
    r"juzgado\s+\d+",                # Juzgados específicos
]
```

### Methods

#### check_response()

Rule-based hallucination detection.

```python
async def check_response(
    self,
    response: str,
    context: str,
    question: str
) -> HallucinationCheckResult
```

**Parameters:**
- `response` (str): LLM-generated response to validate
- `context` (str): Context provided to the LLM
- `question` (str): Original user question

**Returns:**
- `HallucinationCheckResult`: Result with `is_valid`, `confidence`, `flags`, `explanation`

**Detection Logic:**

1. **System Access Claims** (-0.3 confidence)
   ```python
   for pattern in self.compiled_patterns:
       if pattern.search(response):
           flags.append("claims_system_access")
           confidence -= 0.3
   ```

2. **Invented Specific Data** (-0.4 confidence)
   ```python
   for pattern in self.data_patterns:
       matches = pattern.findall(response)
       for match in matches:
           if match in context or match in question:
               continue  # Data is from user/context, not invented
           flags.append(f"invents_specific_data:{match}")
           confidence -= 0.4
   ```

3. **Unverified URLs** (-0.2 confidence)
   ```python
   url_pattern = re.compile(r"https?://[^\s]+")
   urls = url_pattern.findall(response)
   if urls:
       flags.append("mentions_urls")
       confidence -= 0.2
   ```

4. **Unknown Proper Nouns** (-0.1 confidence each)
   ```python
   known_places = ["san rafael", "mendoza", "argentina", "defensoría"]
   capitalized_words = re.findall(r"\b[A-Z][a-z]+\s+[A-Z][a-z]+\b", response)
   for word in capitalized_words:
       if word.lower() not in known_places and word not in context:
           flags.append(f"unknown_proper_noun:{word}")
           confidence -= 0.1
   ```

5. **Appropriate Uncertainty** (+0.1 confidence)
   ```python
   uncertainty_count = sum(1 for ind in self.UNCERTAINTY_INDICATORS if ind in response.lower())
   if uncertainty_count > 0:
       confidence += 0.1  # Bonus for expressing uncertainty
   ```

6. **Excessive Length** (-0.1 confidence)
   ```python
   if len(response.split()) > 300:
       flags.append("excessive_length")
       confidence -= 0.1
   ```

7. **Answers Impossible Questions** (-0.5 confidence)
   ```python
   impossible_questions = [
       r"cuánto (tiempo|demora|tarda)",
       r"cuándo (va a|se va a)",
       r"qué (día|fecha) exacta"
   ]
   for pattern_str in impossible_questions:
       if re.search(pattern_str, question.lower()):
           if any(p.search(response) for p in self.data_patterns[:2]):
               flags.append("answers_impossible_question")
               confidence -= 0.5
   ```

**Final Validation:**

```python
confidence = max(0.0, min(1.0, confidence))
is_valid = confidence >= 0.6 and not any("invents" in f for f in flags)
```

## Performance Considerations

### Rule-Based Validation

- **Latency**: < 10ms (synchronous regex matching)
- **Use case**: Real-time validation in production

### LLM-Based Validation

- **Latency**: 500-2000ms (LLM API call)
- **Use case**: High-stakes responses, audit trail, offline analysis
- **Model**: `glm-4.6:cloud` (specialized for hallucination detection)

### Hybrid Approach

```python
# Fast rule-based check first
rule_result = await detector.check_response(response, context, question)

if rule_result.confidence < 0.8:  # Borderline case
    # Confirm with LLM validator
    llm_result = await detector.check_with_llm(response, context, question)
    return llm_result
else:
    return rule_result
```
