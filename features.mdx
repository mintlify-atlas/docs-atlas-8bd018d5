# Platform Features

The Defensoría Civil divorce platform combines conversational AI, document intelligence, and legal validation into a cohesive system. This page details each feature with real implementation examples.

## WhatsApp Conversational Flow

### Intelligent State Machine

The platform implements a **finite state machine** that guides users through divorce filing steps. Each phase transitions based on validated user input.

    ```python
    async def _phase_cuit(self, case, text: str) -> str:
        import re
        
        # Limpiar el CUIT: quitar guiones y espacios
        cuit_clean = re.sub(r'[\s-]', '', text.strip())
        
        # Validar formato CUIT: 10 u 11 dígitos
        if not re.match(r'^\d{10,11}$', cuit_clean):
            return "El CUIT/CUIL ingresado no es válido..."
        
        # Extraer DNI del CUIT (dígitos 3 al 10)
        dni = cuit_clean[2:10]
        
        # Formatear CUIT con guiones para visualización
        cuit_formatted = f"{cuit_clean[0:2]}-{dni}-{cuit_clean[10]}"
        
        case.cuit = cuit_formatted
        case.dni = dni  # Auto-extracted!
        case.phase = "fecha_nacimiento"
        self.cases.update(case)
        
        return f"✅ CUIT/CUIL: {cuit_formatted}\nDNI extraído: {dni}..."
    ```

    The system **automatically extracts DNI** from CUIT/CUIL format.

    async def search_episodic_memory(self, case_id: int, query: str, limit: int = 5):
        # Generar embedding de la query
        embeddings = await self.llm.embed([query])
        query_embedding = embeddings[0]

### Contextual Prompt Building

All memory types combine into rich LLM context (`memory_service.py:218-247`):

```python
async def build_context_for_llm(self, case_id: int, current_question: str) -> str:
    context_parts = []
    
    # 1. Memoria inmediata (últimos mensajes)
    immediate = await self.retrieve_immediate_memory(case_id)
    if immediate:
        context_parts.append("## Conversación reciente:\n" + "\n".join(immediate))
    
    # 2. Datos de sesión
    session_data = await self.retrieve_session_data(case_id)
    if session_data:
        import json
        context_parts.append(f"## Datos del caso:\n{json.dumps(session_data, indent=2)}")
    
    # 3. Memoria episódica relevante
    episodic = await self.search_episodic_memory(case_id, current_question, limit=3)
    if episodic:
        episodes = "\n".join([f"- {ep['content']}" for ep in episodic])
        context_parts.append(f"## Conversaciones anteriores relevantes:\n{episodes}")
    
    # 4. Conocimiento legal relevante
    knowledge = await self.search_semantic_knowledge(current_question, limit=2)
    if knowledge:
        docs = "\n\n".join([f"**{k['title']}**\n{k['content']}" for k in knowledge])
        context_parts.append(f"## Conocimiento legal aplicable:\n{docs}")
    
    return "\n\n".join(context_parts)
```

### Hallucination Detection

Implemented in `hallucination_detection_service.py:1-100`, the platform **prevents AI from inventing facts**:

<Steps>
  <Step title="Pattern Detection">
    Flags suspicious phrases that indicate fabricated data:
    
    ```python
    HALLUCINATION_PATTERNS = [
        r"según mi base de datos",
        r"en mi sistema",
        r"tengo acceso a",
        r"puedo ver que",
        r"según mis registros",
        r"he verificado que",
    ]
    ```
  </Step>

  <Step title="Data Invention Check">
    Detects specific data not present in context:
    
    ```python
    SPECIFIC_DATA_PATTERNS = [
        r"\d{7,8}",  # DNI
        r"\d{2}/\d{2}/\d{4}",  # Fechas específicas
        r"expediente\s+n[úu]mero\s+\d+",  # Números de expediente
        r"juzgado\s+\d+",  # Juzgados específicos
    ]
    ```

    Only flags data **NOT found in context or user question**.
  </Step>

  <Step title="Confidence Scoring">
    Assigns confidence score (0.0-1.0) based on detected issues:
    
    ```python
    async def check_response(self, response: str, context: str, question: str):
        flags = []
        confidence = 1.0
        
        for pattern in self.compiled_patterns:
            if pattern.search(response):
                flags.append("claims_system_access")
                confidence -= 0.3
        
        for pattern in self.data_patterns:
            matches = pattern.findall(response)
            for match in matches:
                if match in context or match in question:
                    continue  # OK, from user input
                flags.append(f"invents_specific_data:{match}")
                confidence -= 0.4
    ```
  </Step>

  <Step title="Rejection & Logging">
    Invalid responses trigger fallback message:
    
    ```python
    if not hallucination_check.is_valid:
        logger.warning(
            "hallucination_detected",
            case_id=case.id,
            confidence=hallucination_check.confidence,
            flags=hallucination_check.flags
        )
        reply = "Disculpá, tuve un problema. ¿Podés reformular tu consulta?"
    ```
  </Step>
</Steps>

## LLM Router & Fallback System

The **LLMRouter** (`router.py:15-100`) implements intelligent model selection:

```python
class LLMRouter(LLMClient):
    def __init__(self):
        self.providers = {
            'ollama_cloud': OllamaCloudClient(),
            'ollama_local': OllamaClient(),
            'gemini': GeminiClient()
        }
        
        self.model_map = {
            'chat': settings.llm_chat_model,
            'reasoning': settings.llm_reasoning_model,
            'hallucination_check': settings.llm_hallucination_model,
            'vision_ocr': settings.llm_vision_model,
            'embeddings': settings.llm_embedding_model
        }
        
        self.fallback_order = ['ollama_cloud', 'ollama_local', 'gemini']
```

**Task-Specific Routing**:
- **Chat**: General conversation (Ollama Cloud preferred)
- **Reasoning**: Complex legal logic
- **Hallucination Check**: Response validation
- **Vision OCR**: Document extraction (Gemini Vision)
- **Embeddings**: Semantic search (text-embedding-004)

**Automatic Fallback**: If primary provider fails, automatically tries next in chain.
