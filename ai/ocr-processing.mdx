## Overview

The platform uses **vision language models** to extract structured data from Argentine legal documents (DNI, marriage certificates, ANSES certifications). This enables automated form filling and data validation.

```
┌─────────────────────────────────────────────────────────────┐
│                    OCR PIPELINE                             │
├─────────────────────────────────────────────────────────────┤
│  1. Image Upload (Frontend)                                 │
│       ↓                                                      │
│  2. Provider Selection (qwen3-vl:cloud primary)             │
│       ↓                                                      │
│  3. Vision Model Extraction (Gemini fallback)               │
│       ↓                                                      │
│  4. JSON Parsing & Validation                               │
│       ↓                                                      │
│  5. Confidence Scoring & Error Reporting                    │
│       ↓                                                      │
│  6. Session Memory Storage                                  │
└─────────────────────────────────────────────────────────────┘
```

## Document Types

### 1. DNI (Documento Nacional de Identidad)

Argentine national ID card (front and back).

#### Extraction Schema

```python
{
  "numero_documento": "12345678",       # 7-8 digits
  "nombre_completo": "JUAN PEREZ",      # Full name in UPPERCASE
  "fecha_nacimiento": "15/03/1985",     # DD/MM/YYYY
  "sexo": "M",                          # M or F
  "fecha_emision": "10/01/2020"         # DD/MM/YYYY
}
```

#### Implementation

```python
# backend/src/infrastructure/ocr/ocr_service_impl.py:135
async def extract_dni_data(self, image_bytes: bytes) -> OCRResult:
    """Extrae datos estructurados de un DNI argentino"""
    
    prompt = """Eres un experto en extraer datos de documentos argentinos.
Analiza esta imagen de DNI argentino y extrae EXACTAMENTE los siguientes datos en formato JSON:

{
  "numero_documento": "string con 7-8 dígitos",
  "nombre_completo": "string con nombre y apellido",
  "fecha_nacimiento": "DD/MM/AAAA",
  "sexo": "M o F",
  "fecha_emision": "DD/MM/AAAA"
}

Reglas importantes:
- Si algún dato NO está visible o legible, usa null
- Formato de fechas SIEMPRE DD/MM/AAAA
- Número de documento sin puntos ni espacios
- Nombre completo en MAYÚSCULAS como aparece en el DNI

Responde SOLO con el JSON, sin explicaciones adicionales."""
    
    try:
        # Primary: Ollama Vision
        raw_text = await self.vision_client.analyze_image(
            image_bytes, 
            prompt, 
            model=settings.llm_vision_model  # qwen3-vl:cloud
        )
        data = self._parse_json_response(raw_text)
        errors, confidence = self._validate_dni_data(data)
        
        return OCRResult(
            success=len(errors) == 0 or confidence > 0.5,
            data=data,
            confidence=confidence,
            errors=errors,
            raw_text=raw_text
        )
        
    except Exception as e:
        # Fallback: Gemini Vision
        if self.gemini_fallback_enabled:
            return await self._extract_dni_gemini_fallback(image_bytes, prompt)
        else:
            return OCRResult(
                success=False,
                data={},
                confidence=0.0,
                errors=[f"Error en procesamiento OCR: {str(e)}"],
                raw_text=None
            )
```

#### Validation

```python
# backend/src/infrastructure/ocr/ocr_service_impl.py:54
def _validate_dni_data(self, data: Dict[str, Any]) -> tuple[list[str], float]:
    errors = []
    confidence = 0.9
    
    # Validate document number (7-8 digits)
    if not data.get("numero_documento") or not re.match(
        r"^\d{7,8}$", 
        str(data.get("numero_documento", ""))
    ):
        errors.append("Número de documento no válido o no detectado")
        confidence -= 0.3
    
    # Validate full name
    if not data.get("nombre_completo"):
        errors.append("Nombre completo no detectado")
        confidence -= 0.2
    
    # Validate birth date format
    if not data.get("fecha_nacimiento") or not re.match(
        r"^\d{2}/\d{2}/\d{4}$",
        str(data.get("fecha_nacimiento", ""))
    ):
        errors.append("Fecha de nacimiento no válida")
        confidence -= 0.2
    
    return errors, max(0.0, confidence)
```

**Confidence Scoring:**
- Base: 90%
- Missing document number: -30%
- Missing name: -20%
- Invalid birth date: -20%
- **Success threshold:** >50% confidence

def _validate_marriage_data(self, data: Dict[str, Any]) -> tuple[list[str], float]:
    errors = []
    confidence = 0.9
    
    # Validate marriage date
    if not data.get("fecha_matrimonio") or not re.match(
        r"^\d{2}/\d{2}/\d{4}$",
        str(data.get("fecha_matrimonio", ""))
    ):
        errors.append("Fecha de matrimonio no válida")
        confidence -= 0.4
    
    # Validate spouse names
    if not data.get("nombre_conyuge_1") or not data.get("nombre_conyuge_2"):
        errors.append("Nombres de cónyuges incompletos")
        confidence -= 0.4
    
    # Validate location

        errors.append("Lugar de matrimonio no detectado")
        confidence -= 0.1
    
    return errors, max(0.0, confidence)

## Data Validation Against Declared Data

Cross-validate OCR extraction with user-declared data:

```python
class DocumentValidationService:
    async def validate_dni_against_case(self, case_id: int, ocr_result: OCRResult):
        """Valida DNI extraído contra datos declarados por el usuario"""
        
        # Get declared data from session memory
        session_data = await memory_service.retrieve_session_data(case_id)
        declared_dni = session_data.get("dni_declarado")
        declared_name = session_data.get("nombre_declarado")
        
        discrepancies = []
        
        # Check DNI match
        extracted_dni = ocr_result.data.get("numero_documento")
        if declared_dni and extracted_dni != declared_dni:
            discrepancies.append(
                f"DNI no coincide: declarado {declared_dni}, extraído {extracted_dni}"
            )
        
        # Check name match (fuzzy)
        extracted_name = ocr_result.data.get("nombre_completo")
        if declared_name and not self._names_match(declared_name, extracted_name):
            discrepancies.append(
                f"Nombre no coincide: declarado {declared_name}, extraído {extracted_name}"
            )
        
        if discrepancies:
            logger.warning(
                "ocr_validation_discrepancies",
                case_id=case_id,
                discrepancies=discrepancies
            )
            # Flag case for manual review
            await flag_service.create_flag(
                case_id=case_id,
                flag_type="data_mismatch",
                details=discrepancies
            )
        
        return len(discrepancies) == 0
    
    def _names_match(self, name1: str, name2: str) -> bool:
        """Fuzzy name matching (ignoring accents, order)"""
        from difflib import SequenceMatcher
        similarity = SequenceMatcher(None, name1.upper(), name2.upper()).ratio()
        return similarity > 0.85  # 85% similarity threshold
```

## Error Handling

### Common Errors

| Error Type | Cause | Recovery |
|------------|-------|----------|
| `JSONDecodeError` | Model returned non-JSON | Retry with stricter prompt |
| `ValidationError` | Missing required fields | Lower confidence, flag for review |
| `ProviderTimeout` | Vision API slow | Automatic fallback to Gemini |
| `ImageTooLarge` | File size >10MB | Resize and retry |
| `ImageCorrupted` | Invalid image format | Return error to user |

### Latency

- **Ollama Vision:** ~2-3 seconds (primary)
- **Gemini Vision:** ~1-2 seconds (fallback)
- **Total with fallback:** ~5 seconds worst-case

## Best Practices

1. **Image Quality** - Request high-resolution scans (min 300 DPI)
2. **Lighting** - Good lighting reduces errors by ~30%
3. **Orientation** - Auto-rotate images before OCR
4. **File Format** - Prefer JPEG/PNG, avoid PDFs (extract pages first)
5. **Validation** - Always cross-validate with declared data
6. **Confidence Thresholds** - Adjust per document type
7. **Manual Review** - Flag cases with &lt;70% confidence
