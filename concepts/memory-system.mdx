# Memory System

The Defensoría Civil platform implements a **four-tier memory system** to maintain conversational context and provide intelligent, context-aware responses.

## Overview

The memory system combines short-term and long-term memory with semantic search capabilities:

```
┌─────────────────────────────────────────┐
│  Immediate Memory (Last 10 messages)    │  ← Short-term
├─────────────────────────────────────────┤
│  Session Memory (Current case data)     │  ← Structured data
├─────────────────────────────────────────┤
│  Episodic Memory (Past conversations)   │  ← Long-term with vectors
├─────────────────────────────────────────┤
│  Semantic Memory (Legal knowledge)      │  ← Domain knowledge with vectors
└─────────────────────────────────────────┘
```

## Memory Service Architecture

**Location:** `backend/src/application/services/memory_service.py`

```python
class MemoryService:
    """
    Advanced contextual memory service
    Implements 4 types of memory:
    - Immediate: Last 10 messages
    - Session: Current conversation data
    - Episodic: Past conversation history
    - Semantic: Structured legal knowledge
    """
    
    def __init__(self, db: Session, llm: Optional[LLMRouter] = None):
        self.db = db
        self.memory_repo = MemoryRepository(db)
        self.llm = llm or LLMRouter()
```

## 1. Immediate Memory

**Purpose:** Store the most recent conversation context (last 10 messages) for immediate recall.

**Storage:** PostgreSQL with automatic pruning

**Lifespan:** Current conversation session

### Storing Immediate Memory

```python
async def store_immediate_memory(self, case_id: int, content: str):
    """Stores immediate memory (last messages)"""
    self.memory_repo.add_memory(case_id, "immediate", content)
    
    # Keep only last 10
    memories = self.db.query(Memory).filter(
        Memory.case_id == case_id,
        Memory.kind == "immediate"
    ).order_by(Memory.created_at.desc()).all()
    
    if len(memories) > 10:
        for mem in memories[10:]:
            self.db.delete(mem)
        self.db.commit()
```

### Retrieving Immediate Memory

```python
async def retrieve_immediate_memory(self, case_id: int) -> List[str]:
    """Retrieves immediate memory (last 10 messages)"""
    memories = self.db.query(Memory).filter(
        Memory.case_id == case_id,
        Memory.kind == "immediate"
    ).order_by(Memory.created_at.desc()).limit(10).all()
    
    return [m.content for m in reversed(memories)]
```

### Usage Example

```python
# Store user message
await self.memory.store_immediate_memory(case.id, f"Usuario: {text}")

# Store assistant response
await self.memory.store_immediate_memory(case.id, f"Asistente: {reply}")

# Retrieve conversation history
recent_messages = await self.memory.retrieve_immediate_memory(case.id)
# ["Usuario: Hola", "Asistente: ¡Hola! Soy tu asistente...", ...]
```

## 2. Session Memory

**Purpose:** Store structured data about the current case (user info, divorce details, etc.)

**Format:** JSON key-value pairs

**Lifespan:** Entire case lifecycle

### Storing Session Data

```python
async def store_session_memory(self, case_id: int, key: str, value: Any):
    """Stores session data as JSON"""
    import json
    content = json.dumps({key: value})
    
    # Update or create
    existing = self.db.query(Memory).filter(
        Memory.case_id == case_id,
        Memory.kind == "session",
        Memory.content.like(f'%"{key}":%')
    ).first()
    
    if existing:
        data = json.loads(existing.content)
        data[key] = value
        existing.content = json.dumps(data)
        self.db.commit()
    else:
        self.memory_repo.add_memory(case_id, "session", content)
```

### Retrieving Session Data

```python
async def retrieve_session_data(self, case_id: int) -> Dict[str, Any]:
    """Retrieves all session data"""
    import json
    memories = self.db.query(Memory).filter(
        Memory.case_id == case_id,
        Memory.kind == "session"
    ).all()
    
    result = {}
    for mem in memories:
        try:
            data = json.loads(mem.content)
            result.update(data)
        except:
            pass
    
    return result
```

### Usage Example

```python
# Store structured data about children
await self.memory.store_session_memory(case.id, "hijos_total", 2)
await self.memory.store_session_memory(case.id, "hijos_index", 0)

# Retrieve session state
data = await self.memory.retrieve_session_data(case.id)
total = data.get("hijos_total", 0)  # 2
index = data.get("hijos_index", 0)  # 0
```

## 3. Episodic Memory

**Purpose:** Store summaries of past conversations with **vector embeddings** for semantic search.

**Technology:** PostgreSQL + **pgvector** extension

**Embedding Model:** OpenAI `text-embedding-3-small` (via LLMRouter)

### Storing Episodic Memory with Embeddings

```python
async def store_episodic_memory(self, case_id: int, summary: str):
    """Stores episodic summary with embedding for semantic search"""
    # Generate embedding
    embeddings = await self.llm.embed([summary])
    embedding = None
    if embeddings and embeddings[0]:
        emb = embeddings[0]
        # Ensure 1D vector
        if isinstance(emb, list) and len(emb) > 0 and isinstance(emb[0], list):
            emb = emb[0]
        embedding = emb
    
    mem = Memory(
        case_id=case_id,
        kind="episodic",
        content=summary,
        embedding=embedding  # pgvector column
    )
    self.db.add(mem)
    self.db.commit()
    logger.info("episodic_memory_stored", case_id=case_id)
```

### Semantic Search in Episodic Memory

**Uses pgvector's Euclidean distance operator `<->`**

```python
async def search_episodic_memory(
    self, 
    case_id: int, 
    query: str, 
    limit: int = 5
) -> List[Dict[str, Any]]:
    """Semantic search in episodic memory using embeddings"""
    
    # Generate query embedding
    embeddings = await self.llm.embed([query])
    if not embeddings or not embeddings[0]:
        return []
    
    query_embedding = embeddings[0]
    
    # Flatten if nested
    if isinstance(query_embedding[0], list):
        query_embedding = query_embedding[0]
    
    # Vector search using pgvector
    # <-> is Euclidean distance operator
    embedding_str = str(query_embedding).replace(' ', '')
    
    result = self.db.execute(
        text("""
            SELECT id, content,
                   embedding <-> CAST(:query_embedding AS vector) AS distance
            FROM memories
            WHERE case_id = :case_id AND kind = 'episodic'
            ORDER BY distance
            LIMIT :limit
        """),
        {
            "query_embedding": embedding_str,
            "case_id": case_id,
            "limit": limit,
        },
    ).fetchall()
    
    return [
        {"id": r[0], "content": r[1], "distance": r[2]}
        for r in result
    ]
```

### Usage Example

```python
# Summarize and store conversation
summary = await self.memory.summarize_conversation(case.id)
await self.memory.store_episodic_memory(case.id, summary)
# Stored with embedding: "Usuario completó datos personales para divorcio unilateral..."

# Later: semantic search for relevant past conversations
results = await self.memory.search_episodic_memory(
    case.id, 
    "¿qué documentos necesito?",
    limit=3
)
# Returns: Most semantically similar past conversations
```

## 4. Semantic Memory (Legal Knowledge)

**Purpose:** Store structured legal knowledge base with vector embeddings.

**Scope:** System-wide (not case-specific)

**Content:** Legal procedures, requirements, FAQs for divorce in Mendoza

### Data Model

```python
# backend/src/infrastructure/persistence/models.py
class SemanticKnowledge(Base):
    __tablename__ = "semantic_knowledge"
    id = Column(Integer, primary_key=True)
    title = Column(String(255))
    content = Column(Text)
    category = Column(String(64))  # divorcio, hijos, bienes, etc.
    embedding = Column(Vector(1536))  # pgvector
    created_at = Column(DateTime, default=datetime.utcnow)
```

### Searching Legal Knowledge

```python
async def search_semantic_knowledge(
    self, 
    query: str, 
    limit: int = 3
) -> List[Dict[str, Any]]:
    """Search legal knowledge base"""
    
    embeddings = await self.llm.embed([query])
    if not embeddings or not embeddings[0]:
        return []
    
    query_embedding = embeddings[0]
    
    # Flatten if nested
    if isinstance(query_embedding[0], list):
        query_embedding = query_embedding[0]
    
    embedding_str = str(query_embedding).replace(' ', '')
    
    result = self.db.execute(
        text("""
            SELECT id, title, content,
                   embedding <-> CAST(:query_embedding AS vector) AS distance
            FROM semantic_knowledge
            ORDER BY distance
            LIMIT :limit
        """),
        {
            "query_embedding": embedding_str,
            "limit": limit,
        },
    ).fetchall()
    
    return [
        {"id": row[0], "title": row[1], "content": row[2], "distance": row[3]}
        for row in result
    ]
```

### Usage Example

```python

# [
#   {"title": "Plazos del divorcio", "content": "Un divorcio unilateral..."},
#   {"title": "Documentación requerida", "content": "Para iniciar..."}
```

## Building Complete Context for LLM

The `build_context_for_llm` method combines all four memory types:

```python
async def build_context_for_llm(self, case_id: int, current_question: str) -> str:
    """
    Builds complete context for LLM by combining all memory types
    """
    context_parts = []
    
    # 1. Immediate memory (last messages)
    immediate = await self.retrieve_immediate_memory(case_id)
    if immediate:
        context_parts.append(
            "## Conversación reciente:\n" + "\n".join(immediate)
        )
    
    # 2. Session data
    session_data = await self.retrieve_session_data(case_id)
    if session_data:
        import json
        context_parts.append(
            f"## Datos del caso:\n{json.dumps(session_data, indent=2, ensure_ascii=False)}"
        )
    
    # 3. Relevant episodic memory
    episodic = await self.search_episodic_memory(case_id, current_question, limit=3)
    if episodic:
        episodes = "\n".join([f"- {ep['content']}" for ep in episodic])
        context_parts.append(
            f"## Conversaciones anteriores relevantes:\n{episodes}"
        )
    
    # 4. Relevant legal knowledge
    knowledge = await self.search_semantic_knowledge(current_question, limit=2)
    if knowledge:
        docs = "\n\n".join([f"**{k['title']}**\n{k['content']}" for k in knowledge])
        context_parts.append(
            f"## Conocimiento legal aplicable:\n{docs}"
        )
    
    return "\n\n".join(context_parts)
```


## Conversation Summarization

Before storing episodic memory, conversations are summarized:

```python
async def summarize_conversation(self, case_id: int) -> str:
    """Generates conversation summary for episodic memory"""
    immediate = await self.retrieve_immediate_memory(case_id)
    if not immediate:
        return ""
    
    conversation = "\n".join(immediate)
    
    prompt = f"""Resume la siguiente conversación en 2-3 oraciones clave, 
enfocándote en los datos recopilados y el progreso del trámite:

{conversation}

Resumen:"""
    
    summary = await self.llm.chat([{"role": "user", "content": prompt}])
    return summary.strip()
```

## Database Schema

### Memory Table

```sql
CREATE TABLE memories (
    id SERIAL PRIMARY KEY,
    case_id INTEGER NOT NULL REFERENCES cases(id),
    kind VARCHAR(16) NOT NULL,  -- immediate | session | episodic
    content TEXT NOT NULL,
    embedding vector(1536),     -- pgvector extension
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_memories_case_kind ON memories(case_id, kind);
CREATE INDEX idx_memories_embedding ON memories USING ivfflat (embedding vector_cosine_ops);
```

### Semantic Knowledge Table

```sql
CREATE TABLE semantic_knowledge (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    category VARCHAR(64),
    embedding vector(1536),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_semantic_knowledge_embedding ON semantic_knowledge USING ivfflat (embedding vector_cosine_ops);
```

## Vector Search with pgvector

### Installation

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### Distance Operators

- **`<->`** Euclidean distance (L2)
- **`<#>`** Negative inner product
- **`<=>`** Cosine distance

```sql
-- Find 3 most similar memories
SELECT id, content, embedding <-> '[0.1, 0.2, ...]' AS distance
FROM memories
WHERE case_id = 123 AND kind = 'episodic'
ORDER BY distance
LIMIT 3;
```

### Indexing

```sql
CREATE INDEX ON memories USING ivfflat (embedding vector_cosine_ops)
```

### Embedding Caching

Embeddings are stored and reused. No need to re-embed on every query.
