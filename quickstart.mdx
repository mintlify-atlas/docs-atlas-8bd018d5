## Overview

This guide will help you get the Defensoría Civil divorce platform up and running using Docker in less than 5 minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Docker** 20.10+ and **Docker Compose** 2.0+
- **Git** for cloning the repository
- A **Gemini API key** (required for LLM functionality)
  - Get one at [Google AI Studio](https://makersuite.google.com/app/apikey)

## Quick Start

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd def-civil-divorcios
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and add your Gemini API key:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and set your Gemini API key:
    
    ```env
    GEMINI_API_KEY=your_api_key_here
    ```
    
    <Note>
      The `.env.example` file contains sensible defaults for local development. The only **required** change is adding your `GEMINI_API_KEY`.
    </Note>
  </Step>

  <Step title="Start the services">
    Build and start all services using Docker Compose:
    
    ```bash
    docker compose up --build
    ```
    
    This will start:
    - **API Backend** (port 8000)
    - **Celery Worker** (background tasks)
    - **PostgreSQL** with pgvector extension (port 5433)
    - **Redis** (port 7379)
    - **Dashboard** (port 5173)
  </Step>

  <Step title="Verify the installation">
    Check that the API is running:
    
    ```bash
    curl http://localhost:8000/health/
    ```
    
    Expected response:
    ```json
    {"status": "ok"}
    ```
    
    You can also access:
    - **API Documentation**: http://localhost:8000/docs
    - **Dashboard**: http://localhost:5173
  </Step>
</Steps>

## What's Running?

After running `docker compose up`, you'll have the following services available:

| Service | Port | URL | Description |
|---------|------|-----|-------------|
| API Backend | 8000 | http://localhost:8000 | FastAPI REST API |
| Swagger Docs | 8000 | http://localhost:8000/docs | Interactive API documentation |
| Dashboard | 5173 | http://localhost:5173 | Frontend interface |
| PostgreSQL | 5433 | localhost:5433 | Database with pgvector |
| Redis | 7379 | localhost:7379 | Cache and task queue |

<Note>
  Note that PostgreSQL runs on port **5433** (not the standard 5432) and Redis runs on port **7379** (not the standard 6379) to avoid conflicts with local installations.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If you see errors about ports already being in use, you can either:
    
    1. Stop the conflicting service
    2. Change the ports in `docker-compose.yml`
    
    For example, to change the API port from 8000 to 8080:
    ```yaml
    api:
      ports:
        - "8080:8000"  # host:container
    ```
  </Accordion>
  
  <Accordion title="Database connection errors">
    If the API can't connect to the database:
    
    1. Ensure all services are running: `docker compose ps`
    2. Check the database logs: `docker compose logs db`
    3. Verify the `DATABASE_URL` in `.env` matches the docker-compose configuration
  </Accordion>
  
  <Accordion title="LLM API errors">
    If you see errors related to the LLM:
    
    1. Verify your `GEMINI_API_KEY` is set correctly in `.env`
    2. Check API quota at [Google AI Studio](https://makersuite.google.com/app/apikey)
    3. Ensure you have network connectivity to Google's API
  </Accordion>
</AccordionGroup>

## Stopping the Services

To stop all services:

```bash
# Stop and remove containers
docker compose down

# Stop and remove containers AND volumes (deletes all data)
docker compose down -v
```

<Warning>
  Using `docker compose down -v` will delete all database data. Only use this if you want to start completely fresh.
</Warning>
